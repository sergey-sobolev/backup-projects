# Автоматизация для создания резервных копий

## задача 1

сгенерировать скрипт, который будет создавать/обновлять резервные копии проектов на флэшке или удалённой системе
- после успешного сохранения создавать флаг success

брать из yaml файла
- список каталогов для создания резервной копии
- целевой каталог
- параметр - обновлять, создавать копию или создать и скопировать tgz

скрипт должен работать в Linux

использовать rsync для копирования 

добавить скрипт для создания расписания обновления резервных копий

сохраняй изменённые скрипты в git репозитории, используй conventional commits стиль

## задача 2

- сгенерируй документацию по проекту (README.md), добавь примеры использования, требования к системе, инструкции по настройке и установке
- сделай тесты, используя pytest
- сделай setup.py для установки скрипта
- сделай Makefile для подготовки целевой системы к использованию скрипта, включая установку rsync и необходимых для работы скрипта пакетов python
- добавь журнал выполнения операций (log)

## задача 3

- сделай индивидуальные настройки режима создания резервной копии для каждого проекта
- если не указаны индивидуальные, используется режим по умолчанию (default_mode)
- обнови тесты
- имя журнала сделай параметром в общем yaml конфиге
